Questions

What if we don’t use Cypress?

This package is optimized for Cypress to guarantee the 3-day turnaround. If you’re on Playwright, we can quote a parallel package or translate patterns—same stability principles, different runner.

Will this run in our CI?

Yes. We wire an example job for GitHub Actions (or provide Jenkins/Azure DevOps steps), including caching, parallelization, and artifacts (screens/videos/reports).

How do you keep tests from being flaky?

Stable selectors (data-test-ids), explicit waits on app signals, isolated test data, network stubbing where appropriate, and retries used sparingly with root cause triage.

What access do you need?

Staging URL, a temporary test account, and CI access (read is fine for a PR-based integration). We can work via a fork/PR model if preferred. NDA welcome.

Can we get more than 25 tests?

Yep—add-on packs or hourly follow-on support. The starter gives you the foundation + patterns so scaling is easy.
